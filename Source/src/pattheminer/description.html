A benchmark for Pat The Miner

Pat The Miner is an extension to the BeepBeep event stream engine. It provides various functions and processors to perform the following things:

  1. Compute in real time whether a stream of events deviates from a reference "trend"
  2. Compute reference trends from an event stream or a collection of reference event streams
  3. Compute predictions on future events of a stream based on a predefined
  4. Build a predictive function based on past instances of a process
  5. Learn associations between windows of the same trace using a machine learning classifier

This benchmark measures the performance of these functionalities, using various types of input event streams, pattern computations, distance metrics, feature vectors, and learning algorithms.

Structure of the benchmark

The benchmark can be instructed to run experiments of two types:

  • Trend experiments corresponds to throughput measurements on the static trend distance and self-correlated trend distance workflow patterns.
  • Predictive experiments corresponds to throughput measurements on the static prediction and predictive learning and self-correlated prediction workflow patterns.

See information below on how to access these two sets of experiments.

References

The results in this benchmark are mentioned in the following publication:

  • M. Roudjane, R. Khoury, D. Rebaïne, S. Hallé. (2019). Predictive Analytics for Event Stream Processing. Proc. EDOC 2019. Submitted.

Pat The Miner itself has been the subject of the following publications:

  • M. Roudjane, R. Khoury, D. Rebaïne, S. Hallé. (2019). Statistics and Data Mining for Event Stream Analytics. In Information Systems, Elsevier. Submitted.
  • M. Roudjane, R. Khoury, D. Rebaïne, S. Hallé. (2018). Real-Time Data Mining for Event Streams. In Proc. EDOC 2018, IEEE Computer Society, 123-134. DOI: 10.1109/EDOC.2018.00025. This paper uses an earlier version of this lab, hosted on Zenodo.

Prerequisites

The self-contained JAR file is a bundle of the following dependent libraries (which you don't need to install):

  • BeepBeep
  • Pat The Miner
  • Apache Commons Math
  • WEKA

Some experiments involve the R statistical software. The exerpiments are enabled by a command-line switch (--with-r) and will only work if R is installed on the local machine. In addition, R must be setup so that the current user has the following packages installed:

  • evobiR
  • moments
  • stringi
  • stylo

All the logs processed in the lab are either synthetically generated at runtime, or static files located inside the bundled JAR.

How to run this lab

This lab can be run by typing, at the command line:

java -jar pattheminer-benchmark.jar [options]

where options can be:

  • --with-trend: shows experiments for trend computations (corresponds to the experiments in the EDOC 2018 paper mentioned above)
  • --with-pred: show experiments for predictive analytics (corresponds to the experiments in the EDOC 2019 paper mentioned above)
  • --with-r: show experiments that compare R with BeepBeep (corresponds to the experiments in the Information Systems paper mentioned above)
  • --datadir dir: instructs LabPal to write the generated log files to dir (the R experiments read traces from these files)

See the Help page for more information.
